# AGENTS

## Mobile Responsiveness

Mobile responsiveness is a hard acceptance criterion for all UI work.

Any new page, section, component, or layout change must:

- use shared responsive layout primitives where possible
- preserve mobile readability at `375px`, `390px`, and `430px` widths
- avoid horizontal overflow
- ensure fixed or sticky UI does not block content
- include mobile screenshots in the review packet when visual UI changes are made
- update `docs/MOBILE-TODO.md` if follow-up mobile work remains

## Workflow

- Treat `docs/MOBILE-TODO.md` as the primary mobile roadmap.
- Keep changes small, focused, and reviewable.
- Prefer local screenshot review during active development.
- Preserve personality, readability, accessibility, and desktop stability.
